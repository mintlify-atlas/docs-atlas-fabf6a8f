---
title: "StickToBottomOptions — all constructor options"
description: "All constructor options for StickToBottom: required element getters, spring animation defaults, resize and initial scroll behavior, and custom target offsets."
---

`StickToBottomOptions` is the object you pass to `new StickToBottom(options)`. It has two required element getters and a set of optional properties that control the default spring animation, resize behavior, initial scroll behavior, and target position calculation.

```typescript
import { StickToBottom } from 'stick-to-bottom-svelte';

const stickToBottom = new StickToBottom({
  scrollElement: () => scrollElement,
  contentElement: () => contentElement,
  // optional: spring defaults, resize, initial, targetScrollTop
});
```

---

## Required options

<ParamField body="scrollElement" type="() => HTMLElement | null | undefined" required>
  A getter that returns the scroll container element — the element that has `overflow-y: auto` or `overflow-y: scroll`. Wrapping the reference in an arrow function lets the constructor capture the reactive binding correctly in Svelte 5.

  ```svelte
  <script lang="ts">
    let scrollElement = $state<HTMLElement>();

    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
    });
  </script>

  <div bind:this={scrollElement} class="overflow-y-auto">
    ...
  </div>
  ```
</ParamField>

<ParamField body="contentElement" type="() => HTMLElement | null | undefined" required>
  A getter that returns the direct child wrapper element inside the scroll container. The `ResizeObserver` is attached to this element to detect when new content is added. It must be a direct child of `scrollElement`.

  ```svelte
  <div bind:this={scrollElement} class="overflow-y-auto">
    <div bind:this={contentElement}>
      <!-- Messages, logs, or other dynamic content -->
    </div>
  </div>
  ```
</ParamField>

## Spring animation options

These three options set the instance-level defaults for all spring animations. They can be overridden per-call in `scrollToBottom()`. All three are optional — if omitted, the library defaults are used.

<ParamField body="damping" type="number" default="0.7">
  Controls how much the spring oscillation is damped. A value from `0` to `1`. `0` means no damping (the scroll will bounce indefinitely); `1` means critically damped (no overshoot). The default of `0.7` gives a gentle, natural deceleration.

  ```svelte
  <script lang="ts">
    // Less oscillation — scroll settles quickly
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      damping: 0.85,
    });
  </script>
  ```
</ParamField>

<ParamField body="stiffness" type="number" default="0.05">
  Controls how fast the spring accelerates toward its target. A value from `0` to `1`. Higher values mean the scroll accelerates faster. The default of `0.05` gives a smooth, gradual ramp-up.

  ```svelte
  <script lang="ts">
    // Snappier acceleration
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      stiffness: 0.12,
    });
  </script>
  ```
</ParamField>

<ParamField body="mass" type="number" default="1.25">
  The inertial mass of the animated scroll. A value roughly between `0.5` and `3`. Higher mass makes the animation feel heavier and slower to start and stop. The default of `1.25` gives a slightly weighty, natural feel.

  ```svelte
  <script lang="ts">
    // Lighter feel — responds faster
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      mass: 0.8,
    });
  </script>
  ```
</ParamField>

## Behavior options

<ParamField body="resize" type="Animation">
  The animation to use when the content element is resized (i.e., new content is appended). If not provided, the instance-level spring defaults are used. Set to `'instant'` for a no-animation jump — useful for log viewers where you want immediate repositioning.

  ```svelte
  <script lang="ts">
    // Jump instantly when new log lines appear
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      resize: 'instant',
    });
  </script>
  ```

  ```svelte
  <script lang="ts">
    // Use a custom spring for resize events only
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      resize: { damping: 0.6, stiffness: 0.08, mass: 1.0 },
    });
  </script>
  ```
</ParamField>

<ParamField body="initial" type="Animation | boolean" default="true">
  Controls scroll behavior when the component first mounts. `true` (the default) scrolls to the bottom using the instance spring animation. `false` leaves the scroll position unchanged. Passing `'instant'` or a `SpringAnimation` object overrides the animation used for the initial scroll only.

  ```svelte
  <script lang="ts">
    // Don't scroll on mount — start from wherever the user is
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      initial: false,
    });
  </script>
  ```

  ```svelte
  <script lang="ts">
    // Jump to bottom on mount, animate on subsequent resizes
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      initial: 'instant',
      resize: { damping: 0.7, stiffness: 0.05, mass: 1.25 },
    });
  </script>
  ```
</ParamField>

<ParamField body="targetScrollTop" type="GetTargetScrollTop">
  An optional function to override the calculated "bottom" position. By default the bottom is `scrollElement.scrollHeight - 1 - scrollElement.clientHeight`. Provide this function if you want the auto-scroll to stop at a custom offset — for example to account for a pinned input area or to leave a gap at the bottom.

  The function receives the default `targetScrollTop` value and the `ScrollElements` context, and must return a number. The returned value is clamped to `[0, targetScrollTop]`.

  ```svelte
  <script lang="ts">
    // Stop 50px before the actual bottom
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      targetScrollTop: (defaultTarget) => defaultTarget - 50,
    });
  </script>
  ```

  ```svelte
  <script lang="ts">
    // Dynamic offset based on content element height
    const stickToBottom = new StickToBottom({
      scrollElement: () => scrollElement,
      contentElement: () => contentElement,
      targetScrollTop: (defaultTarget, { contentElement }) => {
        const el = contentElement();
        const lastChild = el?.lastElementChild;
        return lastChild
          ? defaultTarget - lastChild.clientHeight / 2
          : defaultTarget;
      },
    });
  </script>
  ```
</ParamField>

## Full example

```svelte
<script lang="ts">
  import { StickToBottom } from 'stick-to-bottom-svelte';

  let scrollElement = $state<HTMLElement>();
  let contentElement = $state<HTMLElement>();

  const stickToBottom = new StickToBottom({
    // Required
    scrollElement: () => scrollElement,
    contentElement: () => contentElement,

    // Spring defaults for all animations
    damping: 0.8,
    stiffness: 0.08,
    mass: 1.2,

    // Instant jump when new content is added
    resize: 'instant',

    // Don't animate on first mount
    initial: false,

    // Always stay 30px above the true bottom
    targetScrollTop: (target) => target - 30,
  });
</script>

<div bind:this={scrollElement} class="h-96 overflow-y-auto">
  <div bind:this={contentElement}>
    <!-- content -->
  </div>
</div>
```
